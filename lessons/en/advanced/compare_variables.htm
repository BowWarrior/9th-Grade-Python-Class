Variable Comparisons

Variables are basically containers that help you and your program remember things.  You use variables when you want to keep track of something.  If you wanted to keep track of how many beepers you picked up, you would use a variable to do this.

Expressions

An expression is a combination of variables, values and operators.

Every expression has a value.

Finding the value of an expression is called evaluating the expression.

The first type of expression we will learn are called Boolean expressions. They are called Boolean expressions, because the value of a Boolean expression is either True or False. Boolean expressions use the following operators,

> 

Greater Than

If the value on the Left is greater than the value on the Right, this expression evaluates to True, otherwise it evaluates to False.

< 

Less Than

If the value on the Left is less than the value on the Right, this expression evaluates to True, otherwise it evaluates to False.

>=

Greater Than or Equal

If the value on the Left is greater than or equal to the value on the Right, this expression evaluates to True, otherwise it evaluates to False.

<=

Less Than or Equal

If the value on the Left is less than the value on the Right, this expression evaluates to True, otherwise it evaluates to False.

==

Equals

If the value on the Left equals the value on the Right, this expression evaluates to True, otherwise it evaluates to False.

!= 

Not Equal To

If the value on the Left is not equal to the value on the Right, this expression evaluates to True, otherwise it evaluates to False.

For example,

5 > 6    evaluates to False
5 < 6    evaluates to True
5 == 6    evaluates to False
5 != 6 evaluates to True
5 <= 6  evaluates to True
5 >= 6 evaluates to False

Boolean Expressions and Variables

The above Boolean expressions were static. Meaning they are always either True or False because the operands (the values 5 and 6 on either side of the operator do not change). We can however use variables in expressions and hence use variables instead of static values.

For example if we have a variable num with a value of 10 and a variable othernum with a value 20.

num <10 evaluates to False (10 is not less than 10)
num<=10 evaluates to True (10 is less than or equal to 10)
num!=10 evaluates to False (10 does not not equal 10)
othernum==num evaluates to False (20 is not equal to 10)
othernum==num*2 evaluates to True (20 is equal to 20)
